    模块化¶

    Python中只有一种模块对象类型

    模块module,指的是Python源代码文件

    包package,指的是模块组织在一起的包和包同名的目录及其相关文件

    模块的四种形式¶

    1. 内置模块:Python解释器自带的模块
    2. pip安装的模块:pip install 模块名
    3. 自定义模块:自定义模块
    4. 包

    导入语句¶

    导入非顶级模块,只将其顶级模块名称加入到本地名词空间中.导入的模块必须使用完全限定名称来访问.

    如果使用as ,as 后的名称直接绑定到导入的模块对象,并将该名称加入到本地名词空间中.

    • 找到from 子句中指定的模块,加载并初始化(不是导入)
    • 对于import子句后的名称
    • 先查from子句导入的模块是否具有该名称的属性
    • 如果没有,则尝试导入该名称子模块中的同名模块
    • 还没找到,抛出Import Error异常
    • 这个名称保存到本地名词空间中,如果使用了as语句,则使用as子句后面的名称

    导入时值检索当前文件夹,如果在子文件夹中的方法,无法直接调用

    自定义模块¶

    1. 模块名就是文件名

    2. 模块名必须符合标识符要求

    3. 不要使用系统模块名,避免冲突

    4. 通常模块名全小写,下划线分割

    模块搜索顺序¶

    从当前→局部→全局→内置

    内存→ 内置→ 自定义

    使用sys.path来查看搜索顺序

    import sys
for p in sys.path:
    print(p)

    显示结果为Python模块的路径搜索顺序

    当加载一个模块时候,需要从这些搜索路径中从前到后一次查找,并不搜索这些目录的子目录,搜索到模块就加载,搜索不到抛异常.

    1. 程序主目录,程序运行的主程序脚本所在的目录
    2. PYTHONPATH目录,环境变量PYTHONPATH设置的目录也是搜索模块的路径
    3. 标准库目录,Python自带的库模块所在的目录
    总结¶

    导入顶级模块,其名称会加入到名称名词空间中,并绑定到其模块对象.

    导入非顶级模块,只将其顶级模块名称加入到本地名词空间中.导入的模块必须使用完全限定名称来访问.

    如果使用了as,as后的名称直接绑定到导入的模块对象,并将该名称加入到本地名词空间中.

    模块运行¶

    import time
#1. 开辟内存空间,内存空间命名为time
# 2. 把time.py中的所有代码读入名称空间,然后运行
# 3. 通过time.方法名使用time模块中的方法

    优点:永不冲突

    缺点:使用麻烦,每次都要从顶级模块开始调用

    from time import sleep
# 1. 开辟内存空间,内存空间命名为time
# 2. 把time.py中的所有代码读入名称空间,然后运行
# 3. 把sleep()读入import和from...import.py中,因此可以直接使用方法名
from time import * #表示导入time下所有公共属性(不以下划线开头的)

    优点:调用省事

    缺点:容易发生冲突

    if __name__ == '__main__':

    上语句常用于模块中功能检测,

    其中,当运行文件为当前文件时,__name__参数默认值为_main_;当调用运行文件外的模块时,__name__参数默值为模块名(默认就意味着可以更改,但是没必要)

    1. 此功能为模块测试功能:测试本模块内部的函数,类
    2. 避免主模块变更的副作用:顶层代码,没有封装,主模块使用没有问题.一旦有了新的主模块,当前模块要被导入,由于源代码没有封装,一并执行
    总结¶
    • 找到from子句中指定的模块,加载并初始化(不是导入)
    • 对于import子句后面的名称
    • 先查from子句导入的模块是否具有该名称的属性
    • 如果不是,尝试导入该名称的子模块
    • 还没找到,抛ImportError异常
    • 这个名称保存到本地名词空间中,如果由as子句,则使用as子句后面的名称

    模块的属性¶

    属性 含义
    __file__ 字符串,源文件路径
    __cached__ 字符串,编译后的字节码文件路径
    __spec__ 显示模块规范
    __name__ 模块名
    __package__ 当模块是包时,同__name__ ;否则,可以设置为顶级模块的空字符串

    包¶

    是个文件夹,内部含有__init__.py 文件(给目录用的),此文件为包属性,有了属性的目录文件夹就是包

    包也是module类型,但是包没有__file__属性

    new中的PythonPackage目录生成时自带__init__,这就是包

    总结¶

    包能够更好的组织模块,方便调用(包→模块→类→方法)

    包目录中__init__.py是在包第一次导入的时候就会执行,内容可以为空,也可以是用于该包初始化工作的代码,最好不要删除(低版本不可删除).

    导入子模块一定会加载父模块,但是导入父模块不一定会导入子模块

    包目录之间只能用.点号作为分隔符,表示模块机器子模块的层级关系

    模块是命名空间,其内部的顶级标识符,都是它的属性,可以通多过__dict__或者dir(module)查看.

    包也是模块,但模块不一定是包,包是特殊的模块,是一种组织方式,它包含__path__

    绝对导入和相对导入¶

    相对导入与绝对导入仅针对包内导入而言.及含有__init__.py文件的目录.

    绝对导入¶

    在import语句或from导入模块,模块名称最前面不是以.点开头的

    绝对导入总是去搜索模块搜索路径中找

    相对导入¶

    只能在包内使用,且只能用在from语句中

    用.点号,表示当前目录内.同级调用

    ..表示上一级目录

    ...表示上上一级

    不要在顶层模块中使用相对导入

    #举例a.b.c模块,c是模块,c的代码中使用 
from . import d # import a.b.d
form .. import e # import a.e
form .d import x # a.b.d.x
form ..e import x # a.e.x

    使用相对导入的模块就是为了内部 互相的引用资源的,不是为了直接运行的,对于包来说,正确的使用方式还是在顶级模块使用这些包.

    使用相对导入就不能当作主模块使用.

    访问控制(模块内变量)¶

    • __all__ 相当于加一个列表或元组,元素是字符串,每一个元素都是一个模块内的变量名,列表内的数据都可以在被调用时访问到

    • 只影响from xxx import * 这种调用方式

    __all__ = ['x','y']

    在自定义模块时,使用_x或者__y相当于时==保护变量==,除非在调用时直接引用变量名,否则不会被dir()调用出来

    总结¶

    一. 使用from xyz import * 导入

    1. 如果模块没有__all__,from xyz import * 只导入非下划线开头的模块变量.如果是包,子模块也不会导入,除非在__all__中设置,或__init__.py中使用相对导入
    2. 如果模块有__all__from xyz import * 只导入__init__ 列表中指定名称,哪怕这个名词时下划线开头或者子模块
    3. from xyz import * 方法导入,使用简单,但是副作用时打入大量不需要使用的变量,甚至会导致遍了变量名的冲突.而__all__ 可以控制被导入模块在这种导入方式下能够提供的变量名称,就是为了阻止from xyz import * 导入过多模块变量.所以编写模块时,尽量加入__all__
    4. 导入包时,默认调用__all__=[]列表中默认是非下划线开头的变量,被导包时会被直接调用,可以通过__all__方法来控制包中的变量名的管理.
