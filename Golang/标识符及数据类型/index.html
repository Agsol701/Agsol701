    标识符¶

    标识符概念¶

    Golang对各种变量,方法等命名时使用的字符序列称之为标识符.

    命名规则¶

    1. Go语言中标识符由字母,数字,下划线组成,且数字不能开头.

    2. 严格区分大小写.

    3. _下划线在Go中是一个特殊的标识符,成为空标识符,可以代表任何其他的标识符,但是它对应的值会被忽略.所以只能作为占位符使用,不能作为标识符使用.

    4. 不能以系统保留关键字(一共25个)作为表示符,比如break,if 等

    5. 文件名推荐使用下划线命名方式

    注意事项¶

    1. 包名:保持package的名字尽量和目录文件夹保持一致,尽量采用有意义的包名,简短,不要和标准库冲突.
    2. 变量名,函数名,常量名:采用驼峰体
    3. 如果变量名,函数名,常量名首字母大写,则可以被其他的包访问;如果首字母小写,则只能在本包中使用(注意:可以简单理解成,首字母大写时公开的,首字母小写时私有的,),golang中没有public,private关键字.

    关键字¶

    关键字始指编程语言中预先定义好的具有特殊意义的标识符,关键字和保留字都不建议用作变量名.

    Go有25个保留关键字和37个预定义标识符,其中包括基础数据类型和系统内嵌函数.

    25个保留关键字,不能用于自定义变量名,只能在特定语法中使用.

        break        default      func         interface    select
    case         defer        go           map          struct
    chan         else         goto         package      switch
    const        fallthrough  if           range        type
    continue     for          import       return       var

    以及37个保留关键字,主要对应内建的常量,类型和函数

    Constants:

Types:

    数据类型¶

    基本数据类型¶
    • 数字类型:整数类型(int,int8,int16,int32,int64,uint,uint8,uint16,uint32,uint64,byte) ,浮点型(float32,float64,)

    int8一个字节8位,int16两个字节,一次类推

    uint表示无符号

    byte一个字节

    float32相当于单精度,float64相当于双精度

    • 字符串类型:string(官方归档)

    • 布尔型:true 或 false

    • 字符型:没有专门的字符型,使用byte来保存单个ascii中的字符(一个汉字三个字节,byte存不进去)

    • rune : 就是int32的别名 表示一个Unicode码点,可以存中文

    • complex64 complex128 复数

    派生/复杂类型:¶
    • 指针类型(pointer)
    • 数组类型
    • 结构体类型(struct) 有点像class
    • 管道类型(channel) 多并发时用
    • 函数类型
    • 切片类型(slice) 动态数组
    • 接口类型(interface)
    • map类型 类似集合

    基本数据类型的默认值¶

    go中,数据类型都有一个默认值,当声明变量而未初始化(赋值)时,就会使用默认值.默认值也叫零值.

    没有开辟过内存空间的指针类型.默认值均为nil.

    数据类型 默认值
    整型 0
    浮点型 0.000..
    字符串 ""
    布尔类型 false

    nil:指针,map,slice,

    结构体是一个值类型,其零值是属性的零值.

    基本数据类型的转换¶

    go在不同类型的变量之间赋值时需要显示转转换,也就是不能自动转换.(高低精度之间的转换也不可以)

    //下面会报错
var i int32 = 100
var j float32 = i
//这样就不会报错
var j float32 = float32(i)

    注意:

    1. Go中,数据类型的转换可以从表示范围小的→表示范围大,也可以从范围大→范围小
    2. 被转换的时==变量存储的数据==(即值),变量本身的数据类型并没有变化.
    3. 转换中,将int64转换成int8,编译时不会报错,只是转换的结果时按溢出处理,与期望值不同.所以在转换时,一定要注意范围.
    //错题1
func main(){
    var n1 int32 = 12
    var n2 int64 
    var n3 int8

    n2 = n1 + 10  //报错:n1时int32,10也会自动编程int32,和也为int32,但是n2是int64,所以报错
    n3 = n1 + 20//同理
}
//改正
n2 = int64(n1) + 10
n3 = int64(n1) + 20


//错题2
func main(){
    var n1 int32 = 12
    var n3 int8
    var n4 int8
    n4 = int8(n1) + 127 //编译没问题,但是最终结果会溢出
    n3 = int8(n1) + 128 // 128超出int8,编译不过
}

    值类型和引用类型¶

    常见的值类型和引用类型¶

    值类型¶

    基本数据类型int系列,float系列,bool,string,数组和结构体struct

    引用类型¶

    指针,切片slice,map,管道channel,interface等都是引用类型

    使用特点¶

    值类型¶

    变量直接存储值,内存==通常==在栈中分配

    引用类型¶

    变量存储的是一个地址,这个地址对应的空间才是真正存储数据值,内存==通常==在堆上分配,当没有任何变量引用这个地址时,该地址对应的数据空间就成为一个垃圾,由GC来回收.

    golang中存在逃逸分析,所以上述为通常情况,也会出现值类型跑到堆中或者引用类型跑到栈中的情况.

    数据类型和内存管理¶

    go中存在逃逸分析,所以没有绝对的数据存放在堆区还是栈区的说法.

    堆区和栈区的分配,不是物理上的分配,而是逻辑上的分配

    go将内存在逻辑上分为三块区域(栈区,堆区,代码区)

    基本数据类型一般分配到栈区

    引用数据类型一般分配到堆区(map,切片)

    代码本身存放在代码区

    自定义数据类型¶

    为了简化数据类型定义,Go支持自定义数据类型

    基本语法¶

    type 自定义数据据类型名  数据类型  //可以理解为 一个别名
type myint int  //这时myint 就等价于int来使用了,相当于都是int类型,但是go认为这是两种数据类型
type mySum func(int,int) int  //这时mySum就等价一个函数类型func(int ,int) int

    注意:这时的myint和int在go中还是两个数据类型,

    通过type关键字的定义,MyInt就是一种新的类型,具有int的特性.

    //证明
type myInt int
var num1 myInt
var num2 int
num1 = 40
num2 = num1  //这里会报错哦,因为go认为myInt和int不是同一种数据类型,不能赋值
fmt.Println("num1 = ",num1)
//如何改:强制类型转换
num2 = int(num1)

    类型别名¶

    Go1.9版本新增的功能.

    类型别名的规定:TypeAlias只是Type的别名,本质上是同一个类型.

    type TypeAlias = Type

    比如之前的rune和byte就是类型别名

    type byte = uint8
type rune = int32

    类型定义和类型别名的区别¶

    表面上只是差了一个等号=

    type NewInt int
type MyInt = int
func main()  {
    var a NewInt
    var b MyInt
    fmt.Printf("type of a: %T\n",a)
    fmt.Printf("type of b: %T\n",b)
}
>>>
type of a: main.NewInt
type of b: int

    通过结果可以看到,类型定义是生成了main包下的一个新的类型,而类型别名只是在代码中存在,编译完成时不会存在MyInt类型.

    new和make¶

    go中,对于引用类型,在使用的时候不仅要声明,还要为其分配内存空间,否则无法使用.

    而值类型的声明不需要分配内存空间,因为在声明的时候已经默认分配好了内存空间.

    new¶

    new是一个内置函数.

    func new(Type) *Type
    • Type表示类型，new函数只接受一个参数，这个参数是一个类型
    • *Type表示类型指针，new函数返回一个指向该类型内存地址的指针。

    使用new函数得到的是一个类型的指针，并且该指针对应的值为该类型的零值。

    func main() {
    a := new(int)
    b := new(bool)
    fmt.Printf("%T\n", a) // *int
    fmt.Printf("%T\n", b) // *bool
    fmt.Println(*a)       // 0
    fmt.Println(*b)       // false
}

    make¶

    make也是用于内存分配的，区别于new，它只用于slice、map以及chan的内存创建，而且它返回的类型就是这三个类型本身，而不是他们的指针类型，因为这三种类型就是引用类型，所以就没有必要返回他们的指针了。

    func make(t Type, size ...IntegerType) Type

    在使用slice、map以及channel的时候，都需要使用make进行初始化，然后才可以对它们进行操作。

    new与make的区别¶

    1. 二者都是用来做内存分配的。
    2. make只用于slice、map以及channel的初始化，返回的还是这三个引用类型本身；
    3. 而new用于类型的内存分配，并且内存对应的值为类型零值，返回的是指向类型的指针。
