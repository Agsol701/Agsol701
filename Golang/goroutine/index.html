    goroutine¶

    进程和线程¶

    1. 进程就是程序.程序在操作系统中的一次执行过程.进程是系统资源分配和调度的基本单位.
    2. 线程是进程的一个执行实例.是程序执行的最小单位.它是比进程更小的可以独立运行的基本单位.
    3. 一个进程可以创建和销毁多个线程.同一个进程中的多个线程可以并发执行
    4. 一个程序至少由一个进程,一个进程至少有一个线程.

    并发和并行¶

    1. 多线程程序在单核上运行,就是并发,
    2. 多线程程序在多核上运行,就是并行.

    go协程和主线程¶

    特点¶
    1. 有独立的栈空间
    2. 共享程序的堆空间
    3. 调度由用户控制
    4. 协程是轻量级的线程
    package main

import (
    "fmt"
    "strconv"
    "time"
)

func test() {
    for i := 1; i <= 10; i++ {
        fmt.Println("test hello world" + strconv.Itoa(i))
        time.Sleep(time.Second)
    }
}

//这种情况下会先将10个test的world执行完毕后再执行主函数的golang
// func main() {
//  test()
//  for i := 1; i <= 10; i++ {
//      fmt.Println("main hello golang" + strconv.Itoa(i))
//      time.Sleep(time.Second)
//  }
// }

func main() {
    go test() //开启了一个协程,main主线程和test协程同时执行
    for i := 1; i <= 10; i++ {
        fmt.Println("main hello golang" + strconv.Itoa(i))
        time.Sleep(time.Second)
    }
}
    说明¶
    1. 如果主线程退出了.即使协程还没有执行完毕,也会退出.
    2. 如果协程在主线程结束之前结束,则自己结束协程.
    3. 主线程是一个物理线程,直接作用与cpu之上.是重量级,非常消耗cpu资源
    4. 协程从主线程开启,是轻量级的线程,是逻辑态.对资源消耗相对较小.
    5. Golang的协程机制是是go的特点,可以轻松开启上万协程.其他语言的并发一般是多线程,相对占用资源多,资源耗费大.

    goroutine调度模型¶

    MPG模式¶

    M:操作系统的主线程(是物理线程)

    P:协程执行需要的上下文

    G:协程

    设置golang运行的cpu数¶

    go1.8之后.默认让程序运行在多核上.

    1.8之前需要通过runtime包实现

    package main
import(
    "runtime"
    "fmt
)
func main() {
    //查看自己cpu个数
    cpuNum := runtime.NumCPU()
    fmt.Println(cpuNum)
    //自己设置使用多少个cpu
    runtime.GOMAXPROCS(cpuNum - 1)
}
