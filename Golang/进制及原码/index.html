    进制¶

    对于整数¶

    二进制¶

    0,1 满2进1

    golang中不能直接用二进制表示一个整数,沿用了c的特点,实在要输入用格式化输出的%b来输出.

    var i int = 5
fmt.Printf("%b",i)
>>>101
    十进制¶

    0~9,满10进1

    八进制¶

    0~7 ,满8进1,以数字0来头表示

    var i int = 011 //表示八进制
fmt.Println(i)
>>>9
    十六进制¶

    0~以及A~F表示,满16进1,以0x或0X开头表示,A~F不区分大小写.

    var i int = 0x11
fmt.Println(i)
>>>17

    十进制转其他进制¶

    二进制转八/十六进制¶

    从右到左将二进制每三/四个以组翻译为8进制

    二-->八
11 010 101 二进制
101 --> 5
010 --> 2
11 --> 3
--> 0325  八进制
----------------------
1101 0101  二进制
0101 --> 5
1101 --> D
--> 0xD5

    八/十六转二同理,每位对应二的三/四位,位不足的用0补全.

    0237二
2 --> 010
3 --> 011
7 --> 111
--> 10011111八

    原码,反码,补码¶

    对于有符号的而言:

    1. 二进制的最高位是符号位:0表示正号,1表示负号.
    2. 正数的原码,反码,补码都一样
    3. 负数的反码=它的原码符号位不变,其他位取反(0->1,1->0)
    4. 负数的补码=它的反码+1
    5. 0的反码,补码都是0
    6. 计算机运算的时候,都是以补码的方式来运算的.
    7. 计算机的存储也是以补码的形式存储的.
    8. 若含负数,计算出的结果依旧是补码,需要翻译为原码再转十进制.

    二进制的溢出和补数¶

    溢出¶

    溢出的本质是计算机无法存放过大或者过小的数据。

    假设一个计算机CPU是4位的，那么每一位或者为0，或者为1，根据排列组合，这四位最多共有2*2*2*2=16种可能的组合方式，也就是说这台计算机只能最多表示16个数字。

    以计算机中的无符号整数为例，那么4位CPU的计算机表示出来的就只有0~15这16个数字。如果你拿两个数，一个为11，另一个为5，做加法的话，计算结果会显示为0而不是16。

    因为11加4已经等于15了，再加1它已经无法表示，所以又回到了0处，这种情况就属于上溢。反之，2-3的话，得到的结果为15，因为2-2已经为0，再减的话就转回到了15，这属于下溢。

    总之一句话，溢出反应了计算机处理能力的上限和下限，太大的数和太小的数均无法直接呈现出来。

    补数¶

    将负数的补码，当成无符号数的原码，即是此负数的同余数，或叫补数。此是补码的真谛，补码即是反码，并不是相反数，而是同余，正是同余，才可以将减变成加。

    比如25217-25216 = 25217+40320 = 65537，对于16位的数值来说，表示范围[-32768，0，32767]，模为65536，因此上溢出，65537 = 1，因此25217-25216 = 1。

    1 -1 = 1 + 255 = 0
