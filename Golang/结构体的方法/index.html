    结构体的方法¶

    Go中的方法是作用在指定的数据类型上的,即和指定的数据类型绑定,因此自定义类型都可以由方法,而不仅仅是struct.

    构造函数¶

    Go语言中没有构造函数,可以自己实现.为了防止结构体比较复杂造成的值拷贝开销太大,这个构造函数返回的是结构体的指针类型.

    type Person struct {
    name string
    city string
    age int8 
}
func newPerson(name,city string, age int8) *Person  {
    return &Person {
        name:name,
        city:city,
        age:age,
    }
}
func main()  {
    p1 := newPerson("agsol","cy",90)//调用构造函数
    fmt.Printf("%#v\n",*p1)
    fmt.Printf("%#v\n",p1)
}
>>>
main.Person{name:"agsol", city:"cy", age:90}
&main.Person{name:"agsol", city:"cy", age:90}

    工厂模式¶

    当导包时候,被导入包中的变量名如果是小写字母开头,那就无法在主包中使用,工厂模式可以解决这个问题.

    普通导包¶

    model.go

    package model
//定义一个结构体
type Student struct {
    Name string
    Score float64
}

    main.go

    package main
import (
    "fmt"
    "gocode/day08/factory/model"
)
func main() {
    var stu = model.Student{
        Name:  "tom",
        Score: 34.9,
    }
    fmt.Println(stu)
}
    工厂模式¶

    当被导入包中变量名以小写字母开头时

    model.go

    package model
//定义一个结构体
type student struct {
    Name string
    Score float64
}
func NewStudent(n string,s float64) *student {
    return &student{
        Name :n,
        Score :s,
    }
}

    main.go

    package main
import (
    "fmt"
    "gocode/day08/factory/model"
)
func main() {
    var stu = model.NewStudent("agsol",99.9)
    fmt.Println(stu,*stu)
}

    注意:如果其他包要是用这个结构体,那么这个结构体的字段也需要是大写,比如上述代码中,如果将Name改为name运行,stu.name,则抛undefine异常.

    stu.score undefined (cannot refer to unexported field or method score)

    方法和接收器¶

    Go语言中的方法(Method)是一种作用于特定类型变量的函数.这中特定类型的变量叫做接收器(Receiver).接收器的概念类似其他语言中的this或者self.

    定义¶
    func (接收器变量receicer 接收器类型type) 方法名methodName (参数列表) (返回参数) {
    函数体
    return 返回值 //非必须
}

    其中:

    • 接收器变量:接收器中的参数变量名在命名时,官方建议使用接收器类型名的第一个小写字母,而不是self或者this之类的命名.比如:Person类型的接收器变量名应该命名为p,Connector类型的接收器变量应该命名为c等.

    接收器变量就相当于一个形参

    • 接收器类型:接收器类型和参数类似,可以是指针类型和非指针类型.

    接收器类型标识了形参的类型

    • 方法名,参数列表,返回参数:具体格式与函数定义相同.
    type Person struct {
    name string
    city string
    age  int8
}
//newPerson 是person的构造函数
func newPerson(name string, age int8) *Person {
    return &Person{
        name: name,
        age:  age,
    }
}
//Dream是Person的一个方法
func (p Person) Dream() {
    fmt.Printf("%s的梦想", p.name)
}
func main() {
    p1 := newPerson("agsol", 20)
    p1.Dream()
}
>>>
agsol的梦想

    方法与构造函数的区别是,函数不属于任何类型,方法属于特定的类型.

    值类型的接收器¶

    当方法作用于值类型接收器时,go语言会在代码运行时将接收器的值赋值一份.在值类型接收器的方法中可以获取接收器的成员值,但是修改操作只针对副本,无法下就该接收器变量本身.

    type Person struct {
    name string
    city string
    age  int8
}
func newPerson(name string, age int8) *Person {
    return &Person{
        name: name,
        age:  age,
    }
}
func (p Person) SetAge2(newAge int8) {
    p.age = newAge
}
func main() {
    p1 := newPerson("agsol", 20)
    fmt.Println(p1.age)
    p1.SetAge2(30)  //等价于(*p1).SetAge2(30)
    fmt.Println(p1.age)
}
>>>
20
20
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
type Person struct{
    Name string
}
func (p Person) test()  {
    p.Name = "jack"
    fmt.Println("in test",p.Name)
}
func main()  {
    var p1 Person
    p1.Name = "tom"
    p1.test()
    fmt.Println("in main",p1.Name)
}
>>>
in test jack
in main tom

    哪个Person实例调用这个方法,p就是是哪个Person实例的副本,这点和函数传参非常相似.

    指针类型的接收器¶

    指针类型的接受者由一个结构体的指针组成,由于指针的特性.调用方法时修改接收器指针的任意成员变量,在方法结束后,修改都是有效的.这种方法就类似其他语言中的this或者self.

    type Person struct {
    name string
    city string
    age  int8
}
//newPerson 是person的构造函数
func newPerson(name string, age int8) *Person {
    return &Person{
        name: name,
        age:  age,
    }
}
//使用指针接收器的方法
func (p *Person) SetAge(newAge int8) {
    p.age = newAge // 等价于(&p).name = newAge
}
func main() {
    p1 := newPerson("agsol", 20)
    fmt.Println(p1.age)
    //调用该方法
    p1.SetAge(30)
    fmt.Println(p1.age)
}
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
type Person struct {
    Name string
    Age  int
}
func (p *Person) test()  {
    p.Age = 30
    fmt.Printf("%v\n",p)
    fmt.Printf("%p\n",p)
}
func main() {
    p1 := Person{"agsol", 19}
    p1.test()
    fmt.Printf("%p\n",&p1)
}
>>>
&{agsol 30}
0xc0000044a0  //指针接收器,是同一个地址的变量
0xc0000044a0
    什么时候要使用指针类型的接收器¶
    1. 需要修改接收器中的值
    2. 接收器时拷贝代价比较大的对象
    3. 保证一致性,如果有某个方法使用了指针接收器,那么其他的方法也应该使用指针接收器.
    4. 相对来说,指针类型的接收器更常用,因为可以提高效率.

    任意类型添加方法¶

    Go语言中,接收器的类型可以是任何类型.不仅仅是结构体,任何类型都可以拥有方法.比如给内置int类型定义自定义一个新的类型,然后为自定义的类型添加方法.

    type MyInt int
func(m MyInt) SayHello(){
    fmt.Println("hello")
}
func main()  {
    var m1 MyInt
    m1.SayHello()
    m1 = 100
    fmt.Printf("%#v %T\n",m1,m1)
}
>>>
hello
100 main.MyInt

    注意:不能给别的包的类型定义方法,只能在包内使用.

    方法的调用和传参机制¶

    方法的调用和传参机制与函数基本一致,不一样的是,在方法调用的时候,会将调用方法的实例当作实参传递给方法.

    说明¶
    1. 在通过一个实例去调用方法时,其调用机制于函数一致.
    2. 不同点:当实例调用方法时,该实例本身也会作为一个参数传递到方法中.(如果实例时值类型,就进行方法的拷贝,如果实例是引用类型,则进行地址拷贝.)

    方法与函数的区别¶

    1. 调用方式不同

    函数的调用方式: 函数名(实参列表)

    方法的调用方式: 变量.方法名(实参列表)

    1. 对于普通函数,接收器为值类类型时.不能将指针类型的数据直接传递,反之亦然.

    2. 对于方法(比如struct的方法),接收器为值类型时,可以直接用指针类型的实例调用方法,反过来也同样可以.

    总结:

    • 不管调用形式如何,真正决定时值拷贝还是地址拷贝,看这个方法与哪个类型绑定.调用的时候传入指针或是值类型,编译器会有优化,自动检测receiver的类型,自定变成其需要的类型.
    • 方法中实际传入的数据类型与接收器数据类型一致,但是传入值类型或者指针类型都可以识别,是go的语法糖.
    • 函数中,按形参的数据类型进行传值,数据类型不一致,报错.

    细节¶

    • ==方法==的访问控制范围的规则,与函数一样.方法名首字母小写,只能在本包访问,方法首字母大写,可以在本包和其他包访问.注意,是方法名!!!

    • 如果一个类型实现了String()这个方法,那么fmt.Println默认会调用这个实例的String()进行输出.而不会输出对象内容

    type Person struct {
  Name string
  Age  int
}
func (p Person) String() string {
  return "aaa"
}
func main() {
  p1 := Person{"agsol", 19}
  fmt.Printf("%#v,%v\n",p1,p1)
  fmt.Println(p1)
}
>>>
main.Person{Name:"agsol", Age:19},aaa
aaa

    练习¶

    1. 编写一个方法算该矩形的面积(可以接收长 len，和宽 width)， 将其作为方法返回值。在 main

    方法中调用该方法，接收返回的面积值并打印。

    type MethodUtils struct {
}
func (mu MethodUtils) areas (m,n float64) float64{
    return m*n
}
func main()  {
    var mu1 MethodUtils
    res := mu1.areas(2.5,4.0)
    fmt.Println(res)
}
>>>
10

    ​ 说明:结构体也可以为空

    1. 编写方法：判断一个数是奇数还是偶数
    type MethodUtils struct {
}
func (mu *MethodUtils) Judge(m int) {
 if m%2 == 0 {
     fmt.Println("even number")
 } else {
     fmt.Println("odd number")
 }
}
func main() {
 var mu1 MethodUtils
 // (&mu1).Judge(8)等价于下面的方式
 mu1.Judge(8)
}
>>>
even number
