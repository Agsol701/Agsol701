    接口Interface¶

    Golang提倡面向接口编程.

    interface是一个引用类型,如果没有初始化就使用,就会输出nil

    接口定义了一个对象的行为规范,只定义规范不实现,由具体的对象来实现规范的细节.

    接口类型¶

    Golang中的接口是一种类型,一种抽象的类型.

    Interface是一组method的集合,是duck-type programming的一种体现.接口做的事情就像是定义一个协议(规则),只要一个动物看起来像鸭子,吃起来像鸭子,那它就是一个鸭子,不关系属性(数据),只关心行为(方法).

    接口定义¶

    接口是一种类型.定义了一组方法,这些方法不需要实现(即不需要由方法体),且==interface 不能包含任何变量==.

    每个接口由数个方法组成,接口的定义格式如下:

    type 接口类型名 interface {
    method1 (参数列表1) 返回值列表1
    method2 (参数列表2) 返回值列表2
    ...
}

    其中;

    • 接口名:使用type将接口定义为自定义的类型名.Go语言的接口在命名时,一般会在单词后面加er,如写操作的接口叫Writer.接口名最好要能突出该接口的类型含义.

    • 方法名:当方法名周梓木时大写且这个接口类型名首字母也是大写的时候,这个方法可以被接口所在的包之外的代码访问.每个method都是一个类型.

    • 参数列表,返回值列表:参数列表和返回值列表中的参数变量名可以省略.比如

    type writer interface {
    Write([]byte) error
}

    这个接口我们不知道可以做什么,但是应该可以通过Write方法做一些事情.

    • 接口里的所有方法都没有方法体.即接口的方法都是没有实现的方法.接口体现了程序设计的多态和高内聚低耦合的思想.

    实现接口的条件¶

    一个对象只要实现了接口中的全部方法.那么就实现了这个接口.

    换句话说,接口就是一个需要实现的方法列表.

    //定义一个Sayer接口
type Sayer interface {
    say()
}
//定义dog和cat两个结构体
type dog struct{}
type cat struct{}
//给dog和cat分别实现say方法来实现Sayer接口
func (d dog) say(){
    fmt.Println("wang")
}
func (c cat) say(){
    fmt.Println("miao")
}

    接口类型变量¶

    接口类型变量可以存储所有实现了该接口的实例.

    package main

import "fmt"

//定义一个鸭子接口(run方法,swim方法)
type DuckInterface interface {
    run()
    swim()
}

//写一个唐老鸭结构体,用来实现接口
type TDuck struct {
    name string
    age int
}

//实现接口,只要结构体绑定了接口中的所有方法,就叫:结构体实现了该接口
func (t TDuck) run() {
    fmt.Println("TDuck run,name:",t.name,)
}
func (t TDuck) swim() {
    fmt.Println("TDuck swim,name:",t.name,)
}

//写一个烤鸭结构体
type KDuck struct {
    name string
    age int
}

//实现接口,只要结构体绑定了接口中的所有方法,就叫:结构体实现了该接口
func (k KDuck) run() {
    fmt.Println("KDuck run,name:",k.name,)
}
func (k KDuck) swim() {
    fmt.Println("KDuck swim,name:",k.name,)
}

func main()  {
    //定义一个鸭子接口
    var d1 DuckInterface
    d1 = TDuck{name:"唐老鸭",age:18}
    var d2 DuckInterface
    d2 = KDuck{"烤鸭",3}
    //唐老鸭的run方法
    d1.run()
    //肉鸭的run方法
    d2.run()

}

    接收器类型实现接口的区别¶

    接收器dog为值类型

    func (d dog) move(){
    fmt.Println("动")
}
func main() {
    var x Mover
    var wangcai = dog{} // 旺财是dog类型
    x = wangcai         // x可以接收dog类型
    var fugui = &dog{}  // 富贵是*dog类型
    x = fugui           // x可以接收*dog类型
    x.move()
}

    接收器为指针类型

    func (d *dog) move() {
    fmt.Println("狗会动")
}
func main() {
    var x Mover
    var wangcai = dog{} // 旺财是dog类型
    x = wangcai         // x不可以接收dog类型
    var fugui = &dog{}  // 富贵是*dog类型
    x = fugui           // x可以接收*dog类型
}
    总结¶
    1. 当接收器为值类型的时候,不管receiver时值类型还是指针类型都可以赋值给接口变量.

    这是go语言中对阵阵类型变量求值的语法糖.fugui内部会自动求值*fugui.

    1. 当接收器类型为指针类型的时候,不能给x传入dog类型的wangcai,此时x只能存储*dog类型的值.

    类型与接口之间的关系¶

    一个类型可以实现多个接口¶

    一个类型可以同时实现多个接口,而接口间彼此独立,不知道对方的实现.

    多个类型实现同一接口¶

    Go语言中不同的类型还可以实现统一接口.

    接口嵌套¶

    接口与接口间可以通过嵌套创造出新的接口.

    type Sayer interface {
    say()
}
type Mover interface {
    move()
}
type animal interface {
    Sayer
    Mover
}

type cat struct {
    name string
}

func (c cat) say() {
    fmt.Println("miao")
}
func (c cat) move() {
    fmt.Println("run")
}
func main() {
    var x animal
    x = cat{name: "tom"}
    x.move()
    x.say()
}
>>>
run
miao

    接口嵌套的时候,同样要满足实现所有接口,不需要指定是哪个接口,但是要实现全部方法.

    空接口¶

    定义¶

    空接口指没有定义任何方法的接口.

    因此任何类型都实现了空接口.

    空接口类型的变量可以存储任意类型的变量.

    接口中没有定义任何需要实现的方法时,该接口就是一个空接口.

    //空接口一般不需要提前定义
type test interface{}

func main()  {
    //定义一个空接口变量t
    var t interface{}
    t = "hello"
    fmt.Println(t)
    t = 111
    fmt.Println(t)
    t = true
    fmt.Println(t)
}
>>>
hello
111
true
    应用¶
    作为函数的参数¶

    使用空接口实现可以接收任意类型的函数参数

    // 空接口作为函数参数
func show(a interface{}) {
    fmt.Printf("type:%T value:%v\n", a, a)
}
    作为map的value¶

    使用空接口实现可以保存任意值的字典.

    // 空接口作为map值
var studentInfo = make(map[string]interface{})
studentInfo["name"] = "agsol"
studentInfo["age"] = 18
studentInfo["married"] = false
fmt.Println(studentInfo)

    接口底层模型¶

    一个接口由两部分组成,动态类型用来存储数据的数据类型,动态值用来存储数据的值.

    类型断言¶

    可用于强制类型转换

    语法格式:`x.(T)

    接口值¶

    一个接口的值(简称接口值)是由一个具体类型和具体类型的值两部分组成的.这两部分分别称为接口的动态类型和动态值.

    var w io.Writer
w = os.Stdout
w = new(bytes.Buffer)
w = nil

    想要判断空接口中的值,这个时候就可以使用类型断言.

    语法格式:x.(T)

    其中​x:表示类型为interface{}的变量;T:表示断言x可能是的类型.

    该语法返回两个参数,第一个参数是x转化为T类型后的变量,第二个值是一个布尔值,若为true表示断言成功,为false表示断言失败.

    即将x转换为T类型,如果可以转换,返回T类型的x和true,否则返回false.

    func main(){
    //定义了一个空接口
    var x interface{}
    // x = "hello world"
    x = 123
    v,ok := x.(string)
    //可以配合if-else来进行断言的判断
    if ok {
        fmt.Println(v)
    }else {
        fmt.Println("false")
    }
    //未知类型的时候,可以使用switch方法来进行断言
    switch v := t.(type) {
    case string:
        fmt.Println("is string",v)
    case bool:
        fmt.Println("is bool",v)
    case int:
        fmt.Println("is int",v)
    default:
        fmt.Println("猜不到了",v)
    }
}
//如果断言正确即ok为true,v为源数据值
//如果断言失败即ok为false,若没使用变量ok,则抛异常

    因为空接口可以存储任意类型值的特点,所以空接口在go语言中使用十分广泛.

    注意:

    只有当两个或两个以上的具体类型必须以相同方式进行处理时才需要定义接口.

    不要为了接口而写接口,那样只会增加不必要的抽象.导致不必要的运行时的损耗.

    类型选择¶

    t.(type)会将接口类型的数据转换成其本身的数据类型

    func test(i interface{}) {
    switch i.(type) {
    case int:
        fmt.Printf("%T\n",i)
    case string:
        fmt.Printf("%T",i)
    }
}

func main()  {
    var a int = 10
    var b string = "agsol"
    test(a)
    test(b)
}
>>>
int
string

    Python与Golang中接口的区别¶

    Python中实现接口通过鸭子类型.自下而上的实现,即先实现了方法,再确定实现了接口

    Golang中实现接口,是自上而下的实现,先规定了需要实现哪些方法,再具体实现接口中的方法.

    实践¶

    1. 实现对Hero结构体切片的排序 sort.Sort(data interface)
    package main

import (
    "fmt"
    "math/rand"
    "sort"
)

//1.声明一个Hero结构体
type Hero struct {
    Name string
    Age  int
}

//2.声明一个Hero结构体切片类型
type HeroSlice []Hero

//3.实现interface接口
func (hs HeroSlice) Len() int {
    return len(hs)
}
func (hs HeroSlice) Less(i, j int) bool {
    return hs[i].Age < hs[j].Age
}
func (hs HeroSlice) Swap(i, j int) {
    temp := hs[i]
    hs[i] = hs[j]
    hs[j] = temp
}
func main() {
    var heroes HeroSlice
    for i := 0; i < 5; i++ {
        hero := Hero{
            Name: fmt.Sprintf("英雄~%d", rand.Intn(100)),
            Age:  rand.Intn(100),
        }
        heroes = append(heroes, hero)
    }
    //排序前的顺序
    for _, v := range heroes {
        fmt.Println(v)
    }
    //调用sort.Sort方法
    sort.Sort(heroes)
    //排序后的顺序
    for _, v := range heroes {
        fmt.Println(v)
    }
}

func TypeJudge(items ...interface{}) {
 for _, x := range items {

     switch x.(type) {
     case bool:
         fmt.Println("bool")
     case float32:
         fmt.Println("float32")
     case float64:
         fmt.Println("float64")
     case int, int32, int64:
         fmt.Println("int")
     case string:
         fmt.Println("string")
     case Student:
         fmt.Println("Student")
     case *Student:
         fmt.Println("*Student")
     }
 }
}

func main() {
 var n1 float32 = 1.1
 var n2 float64 = 2.3
 var n3 int32 = 32
 var name string = "tom"

 stu1 := Student{}
 stu2 := &Student{}
 TypeJudge(n1, n2, n3, name, stu1, stu2)
}
