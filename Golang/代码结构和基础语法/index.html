    语法特性¶

    package main  #定义了一个包的名字  
import "fmt"  #程序需要fmt包,格式化输出函数
func main(){                #func表示要定义一个函数
    fmt.Println("hello word,") #main表名了程序开始执行,
}

    • 非注释第一行一定要声明当前文件属于哪个包

    • 每个可执行函数都要包含main函数,main是一个文件执行的入口

    • fmt.println fmt指包中的println方法 自动\n

    • go中的{}不能单独放在一行,必须跟在函数后
    • 代码函数必须放在函数内部
    • 变量如果定义了而没有使用,会引发编译问题
    • 终端中.go fmt main.go自动格式化代码,目前编译器大部分自动调用
    • 命名使用驼峰体
    • Go源文件以go为扩展名
    • Go语言严格区分大小写
    • Go方法由一条条语句构成,每个语句后不需要分号(Go语言会在每行后自动加分号),加了也能用,但是显得不专业
    • Go编译器是一行行进行编译的,因此写代码一行就写一条,不能多条语句写在一行
    • Go语言定义的变量或者import的包如果没有使用到,代码不能通过编译.

    写¶

    package main //
import "fmt"
func main(){
    fmt.Println("hellow world")
}

    • package main 表示该hello go 文件所在包是main,在go中,每个文件都必须归属于一个包

    • import "fmt" 表示引入一个包,包名为fmt,引入该包后就可以使用fmt中的函数,比如:fmt.Println()

    • func main(){

    }

    func 是一个关键字,表示一个函数,main是函数名,表示是一个主函数,即程序的入口

    • fmt.Println()表示调用fmt包的函数Println出入''hello world"

    运行¶

    1. 通过go build 命令对go文件进行编译,生成exe文件,运行hello.exe文件,

    执行效率高,生产中用,更换环境也可以直接使用

    1. go run 直接运行,执行效率低更换环境必须也有go才能用

    注意事项¶

    1. Go源文件以go为扩展名
    2. Go应用程序的执行入口是main()方法
    3. Go语言严格区分大小写
    4. Go方法由一条条语句构成,每个语句后不需要分号(Go语言会在每行后自动加分号),加了也能用,但是显得不专业
    5. Go编译器是一行行进行编译的,因此写代码一行就写一条,不能多条语句写在一行
    6. Go语言定义的变量或者import的包如果没有使用到,代码不能通过编译.

    Go的转义字符(escape char)¶

    1. \t 一个制表符
    2. \n 换行符
    3. \ 一个\
    4. \" 一个"
    5. \r 一个回车
